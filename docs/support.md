# Support & Help

Where to go when you need help, found a bug, or want to report something.

## Something not working in the app?

Use the **in-app bug report**. It's the fastest path and attaches the
diagnostics we need, with data minimized by design (your browsing domains are
never included). Reports are triaged by the team.

## Found a security vulnerability?

**Don't open a public issue.** Email **security@lavasecurity.app** with a
description, impact, and reproduction steps. See the iOS client's
[security policy](https://github.com/lavasecurity/lavasec-ios/blob/main/SECURITY.md)
for scope and disclosure. _(Address to be confirmed.)_

## Question about the open-source client?

Lava's iOS client is open source. For technical questions, build issues, or
feature ideas, open an issue on
[**lavasecurity/lavasec-ios**](https://github.com/lavasecurity/lavasec-ios/issues).

## Something wrong in these docs?

Every page has a **"Was this page helpful?"** prompt at the bottom — choosing
*"Could be better"* opens a pre-filled issue on
[**lavasecurity/lavasec-doc**](https://github.com/lavasecurity/lavasec-doc/issues).
Corrections and pull requests are welcome.
