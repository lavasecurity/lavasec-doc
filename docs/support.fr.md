# Aide et support

Où aller quand vous avez besoin d'un coup de main, que vous avez trouvé un bug ou que vous voulez nous signaler quelque chose.

## Un souci avec l'app ? {#something-not-working-in-the-app}

Passez par le **rapport de bug intégré à l'app** — c'est le plus rapide, et il joint automatiquement les diagnostics dont on a besoin, avec le minimum de données par conception (les domaines que vous visitez ne sont jamais inclus). Les rapports sont traités par l'équipe.

## Vous avez trouvé une faille de sécurité ? {#found-a-security-vulnerability}

**N'ouvrez pas de ticket public.** Écrivez à **security@lavasecurity.app** avec une description, l'impact et les étapes pour reproduire le problème. Consultez la [politique de sécurité](https://github.com/lavasecurity/lavasec-ios/blob/main/SECURITY.md) du client iOS pour le périmètre et la divulgation. _(Adresse à confirmer.)_

## Une question sur le client open source ? {#question-about-the-open-source-client}

Le client iOS de Lava est open source. Pour les questions techniques, les problèmes de compilation ou les idées de fonctionnalités, ouvrez un ticket sur [**lavasecurity/lavasec-ios**](https://github.com/lavasecurity/lavasec-ios/issues).

## Une erreur dans cette doc ? {#something-wrong-in-these-docs}

Chaque page a en bas un encart **« Cette page vous a-t-elle été utile ? »** — choisir *« Ça pourrait être mieux »* ouvre un ticket pré-rempli sur [**lavasecurity/lavasec-doc**](https://github.com/lavasecurity/lavasec-doc/issues). Les corrections et les pull requests sont les bienvenues.

## Comprendre comment Lava fonctionne {#understanding-how-lava-works}

La plupart des questions du type « est-ce que c'est normal ? » trouvent leur réponse directement dans la doc :

- [Aperçu du produit](product/overview.md) et [Catalogue des fonctionnalités](product/features.md) — ce que Lava fait.
- [Filtrage DNS et listes de blocage](architecture/dns-filtering-and-blocklists.md) — comment se comportent le filtrage et le DNS chiffré.
- [Comptes et sauvegarde zero-knowledge](architecture/accounts-and-backup.md) — ce qui est chiffré et ce que les serveurs peuvent voir.
- [Décisions clés](decisions/key-decisions.md) — pourquoi c'est construit comme ça.
