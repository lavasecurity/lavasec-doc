# 제품 개요

Lava Security에 오신 것을 환영해요. 이 페이지는 문서 전체로 들어가는 입구예요. Lava가 무엇인지, 무엇을 약속하는지, 그리고 더 자세한 내용을 어디서 읽을 수 있는지를 짧고 쉽게 소개해요.

## Lava란

Lava Security는 프라이버시를 가장 먼저 생각하는 iOS 앱이에요. 기기에 있는 [NetworkExtension 패킷 터널](../architecture/ios-client.md)을 통해 DNS를 기기 내부에서 직접 필터링해요. 알려진 위험하거나 원치 않는 도메인을 차단하면서도, 여러분의 인터넷 사용을 Lava의 서버로 보내지 않아요. 패킷 터널(`LavaSecTunnel`, `NEPacketTunnelProvider`)은 각 DNS 요청을 휴대폰에서 직접 분석하고, 요청된 도메인을 컴파일된 메모리 매핑 필터 스냅샷과 대조한 뒤, 허용된 요청만 상위 서버로 전달해요. 여러분의 트래픽이 거쳐 가는 Lava 운영 프록시 같은 건 없어요. 필터링은 여러분의 기기에서 내려지는 로컬 판단이에요.

iOS는 이것을 "VPN"이라고 표시하는데, 앱이 시스템 전체의 DNS를 필터링할 수 있는 유일한 방법이 패킷 터널이기 때문이에요. 하지만 Lava는 트래픽을 우회시키는 것이 아니라 **DNS/차단 목록 필터링**을 해요. 범위를 솔직하게 말하면, Lava는 로컬 DNS 도메인 필터링이지, 모든 악성 도메인이나 URL이 차단된다는 **보장**은 아니에요. Lava는 페이지 경로가 아니라 도메인만 보기 때문에, 대체로 신뢰할 수 있는 호스트 안의 특정 한 페이지만 골라서 차단할 수는 없어요. 또한 온보딩이 끝나는 순간 보호가 자동으로 켜지는 것도 아니에요. 지금 보호가 켜져 있는지는 앱 안의 **Guard** 탭에서 확인하는 게 정확해요.

## 프라이버시 약속

> 모든 DNS 필터링은 기기에서 이루어져요. Lava는 여러분의 인터넷 사용을 자사 서버로 보내지 않고, 여러분이 방문하는 도메인의 흐름을 받지도 않아요. 백엔드에는 카탈로그 메타데이터, 사용자별로 암호화된 식별 불가능한 백업, 그리고 여러분이 보내기로 선택한 익명화된 진단 정보만 보관돼요.

이 문장이 기준이에요. 이 문서의 다른 모든 내용은 이 문장과 일치하도록 쓰여 있어요. 선택 요금제를 결제하더라도 필터링이 서버로 옮겨가거나 Lava가 방문 도메인의 흐름을 받게 되지는 **않아요**. 어떤 기능이 서버와 연결될 때는, 문서에서 무엇이 **전송되지 않는지** 분명히 적어 둬요. 평소의 DNS 요청, 인터넷 사용 기록, 그리고 모든 평문은 기기 안에만 남아요. 전체 그림은 [백엔드와 데이터 모델](../architecture/backend-and-data.md)에서 확인하세요.

## 누구를 위한 앱인가

Lava는 신경 쓰지 않고도 더 안전하게 인터넷을 쓰고 싶은 모든 분을 위해 만들어졌어요. 사용 대상에는 기술에 익숙하지 않은 분들도 일부러 포함했어요. 가족을 위해 보호를 설정하는 부모님, 어르신, 그리고 DNS 같은 건 전혀 생각하고 싶지 않은 모든 분이요. 기본 사용 경험은 그냥 알아서 작동해요. 보호를 켜기만 하면 안전 위주의 차단 목록이 필터링을 시작하고, 계정도 필요 없어요. 동시에, 원하는 분들은 더 깊은 설정(맞춤 차단 목록, 다른 리졸버)에도 접근할 수 있어요.

전체적인 어조는 쉽고, 차분하고, 실용적이에요. 위험은 두려움이 아니라 비유로 표현해요.

## 핵심 원칙

- **프라이버시는 자리매김이지 유료 기능이 아니에요.** 필터링은 로컬 판단이에요. Lava의 백엔드는 의도적으로 최소한으로 유지되며, 여러분의 평소 인터넷 사용 도메인이나 DNS 이벤트 흐름을 받지 않아요. 선택 사항인 계정 백업은 [제로 지식](../architecture/accounts-and-backup.md) 방식이에요. 서버에는 암호문과 비밀이 아닌 봉투 메타데이터만 저장돼요.
- **핵심 보호는 영원히 무료예요.** 보호 스위치, 기본 차단 목록 업데이트, 기본적인 로컬 카운트는 절대 제한되지 않고 계정도 필요 없어요.
- **기기 내에서 처리해요.** 보호 엔진은 전적으로 휴대폰 안에 있어요. DNS 분석, 도메인 평가, 상위 서버로의 전달은 모두 패킷 터널 익스텐션 안에서 이루어지며, iOS의 익스텐션당 약 50 MiB 메모리 한도 안에서 동작해요. 차단 목록은 [원본 URL만 사용하는](../architecture/dns-filtering-and-blocklists.md) 방식을 따라요. 앱이 각 상위 목록을 직접 가져와 로컬에서 분석하고, Lava는 제3자 차단 목록 데이터를 직접 보관하거나 제공하지 않아요.
- **결제는 맞춤 설정만 열어줘요. 기본 안전은 절대 결제로 좌우되지 않아요.** 위협 보호 가드레일은 모든 차단 목록 위에 있는 허용 불가 계층으로, 유료든 무료든 누구도 허용 목록에 넣을 수 없어요. 이것은 판단 우선순위로 강제돼요. **위협 가드레일 > 로컬 허용 목록(허용 예외) > 차단 목록 > 기본 허용.** (이 우선순위 자리는 승인된 SHA-256 해시로 연결되고 무결성이 검증돼요. 현재는 항목이 비어 있는 상태로 출시돼요.) 터널은 `isPaid`를 무시해요.
- **기본은 차분하게, 깊이는 찾는 만큼.** 기본 화면은 조용하고 안심을 주며, Soft Shield Guardian 마스코트와 두려움을 자극하지 않는 문구가 앞에 나서요. 더 풍부하고 기술적인 내용은 찾아보면 볼 수 있지만 절대 강요하지 않아요. 이 "차분한 기본, 찾는 만큼의 깊이" 철학은 **LavaTier** 깊이 모델(Floor / Window / Workshop)로 정리돼 있어요. [디자인 시스템](../design-system/overview.md)을 참고하세요.

## 주요 기능 한눈에 보기

- **로컬 DNS 필터링** — DNS를 분석하고, 각 도메인을 컴파일된 스냅샷과 대조해 평가한 뒤, 허용된 요청을 상위 서버로 전달하는 패킷 터널 엔진이에요. 기기 DNS로의 대체(폴백)도 지원해요. [iOS 클라이언트](../architecture/ios-client.md)와 [DNS 필터링과 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **엄선된 차단 목록, 원본 URL만 사용** — Lava는 상위 목록 URL과 승인된 해시만 공개해요. 기기가 직접 목록 데이터를 가져오고, 검증하고, 분석하며, Lava는 제3자 차단 목록 데이터를 미러링하거나 제공하지 않아요. 출시 기본값은 **Block List Project Phishing + Scam**을 활성화해요(`AppConfiguration.lavaRecommendedDefaults`, `OnboardingDefaults.swift`에 정의됨). GPL 출처(HaGeZi, OISD)는 선택 활성화예요. [DNS 필터링과 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **암호화된 DNS 전송** — DoH(관측용 DoH3 표기 포함), DoT(연결을 모아 재사용하고 갱신), DoQ(요청마다 새 연결)을 지원해요. 세 가지 모두 구현되어 있어요. 기기 DNS(네트워크 자체 리졸버)가 출시 기본값이고, 암호화 프리셋은 선택 활성화예요(`AppConfiguration.lavaRecommendedDefaults`, `Sources/LavaSecCore/OnboardingDefaults.swift`에 정의됨). 기본 제공 리졸버 프리셋(Google / Cloudflare / Quad9의 DoH 및 DoT 변형)은 무료예요. 완전한 맞춤 리졸버만 유료로 열려요. [DNS 필터링과 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **허용 예외(허용 목록)** — 차단 목록에 있더라도 직접 도메인을 추가해 허용할 수 있어요. 그래도 위협 가드레일이 우선해요. [제품 기능 개요](features.md)를 참고하세요.
- **Soft Shield Guardian** — Guard 탭, 라이브 활동, 다이내믹 아일랜드에 등장하는 마스코트로, 7가지 표정 상태로 보호 상태를 표현해요. [디자인 시스템](../design-system/overview.md)을 참고하세요.
- **단계별 맞춤 설정(Lava Security Plus)** — 맞춤 설정을 열어주는 선택형 유료 요금제 하나예요(더 큰 필터 규칙 한도 — 공통 기기 안전 가드레일 아래에서 무료 50만 / Plus 200만 컴파일 규칙 — 더 많은 허용/차단 도메인, 맞춤 차단 목록, 맞춤 DNS 리졸버). Plus도 항상 켜져 있는 가드레일은 절대 우회하지 못해요. 터널은 `isPaid`를 무시해요.
- **선택형 계정과 백업** — Apple 또는 Google 로그인과 함께 종단 간 암호화된([제로 지식](../architecture/accounts-and-backup.md)) 설정 백업 및 복구 문구를 제공해요. 계정 삭제는 직접 할 수 있어요. 선택형 패스키 복구 자리도 **마찬가지로 제로 지식**이에요. 그 키는 인증기의 WebAuthn PRF에서 기기 내에서 파생되며, 서버에 보관되는 비밀은 없어요. 기기 내 정식 출시 준비는 여전히 Associated Domains / AASA 호스팅에 달려 있어요 **(예정)**. 계정은 선택 사항이며, 로그인하지 않아도 보호는 온전히 작동해요.
- **기기 내에서만 보는 활동과 보고서** — 기기 내 차단/허용 카운트, 터널 상태, 그리고 선택형 버그 보고 묶음을 제공해요. 이 정보는 실행 중인 터널이 기기에 보관하는 데이터로 만들어져요. 쉬는 동안에는 비어 있고, 보호하는 동안에는 실시간으로 갱신돼요. 평소의 도메인 기록은 기기를 떠나지 않아요. [제품 기능 개요](features.md)를 참고하세요.

## 플랫폼

- **iOS — 출시됨.** Lava는 현재 iOS 앱이에요. 세 개의 번들이 하나의 App Group(`group.com.lavasec`)을 공유해요. 앱(`com.lavasec.app`), 패킷 터널 익스텐션(`.tunnel`), 위젯(`.widget`)이며, 여기에 공유 소스와 공통 `LavaSecCore` 패키지가 더해져요.
- **Android — 예정.** Android의 `VpnService` 위에 네이티브 Kotlin / Jetpack Compose로 이식하는 작업이 예정돼 있어요. 동일한 프라이버시 약속과, 동등성을 검증한 핵심 필터링 동작을 그대로 가져가요. 아직 Android 앱 코드는 출시되지 않았어요.

안정적인 기능 id와 iOS/Android 계약은 [플랫폼 동등성](platform-parity.md)을 참고하세요.
