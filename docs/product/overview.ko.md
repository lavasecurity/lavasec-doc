# 제품 개요 {#product-overview}

Lava Security에 오신 것을 환영해요. 이 페이지는 문서 모음의 현관과 같아요. Lava가 무엇인지, 무엇을 약속하는지, 그리고 더 자세한 내용을 어디에서 볼 수 있는지를 짧고 쉽게 소개해요.

## Lava란 무엇인가요 {#what-lava-is}

Lava Security는 프라이버시를 가장 중요하게 여기는 iOS 앱이에요. 기기 안에서 동작하는 [NetworkExtension 패킷 터널](../architecture/ios-client.md)을 통해 DNS를 기기 내부에서 직접 필터링하며, 알려진 위험하거나 원치 않는 도메인을 차단해요. 이때 브라우징이 Lava의 서버를 거치지 않아요. 패킷 터널(`LavaSecTunnel`, `NEPacketTunnelProvider`)은 휴대폰에서 각 DNS 쿼리를 분석하고, 요청된 도메인을 컴파일된 메모리 매핑 필터 스냅숏과 대조한 뒤, 허용된 쿼리만 상위 서버로 전달해요. 트래픽이 거쳐 가는 Lava 운영 프록시는 존재하지 않아요. 필터링은 여러분의 기기에서 이루어지는 로컬 판단이에요.

iOS는 이것을 "VPN"이라고 표시하는데, 패킷 터널이 앱이 시스템 전체의 DNS를 필터링할 수 있는 유일한 방법이기 때문이에요. 하지만 Lava는 트래픽 라우팅이 아니라 **DNS/차단 목록 필터링**이에요. 범위에 대해 솔직하게 말씀드릴게요. Lava는 로컬 DNS 도메인 필터링이며, 모든 악성 도메인이나 URL이 차단된다는 **보장은 아니에요**. Lava는 페이지 경로가 아니라 도메인을 보기 때문에, 신뢰할 수 있는 호스트에 있는 하나의 나쁜 페이지를 차단하지는 못해요. 또한 보호 기능은 온보딩이 끝나는 순간 자동으로 켜지지 않아요. 보호가 현재 활성화되어 있는지는 앱 내 **Guard** 탭이 가장 정확한 기준이에요.

## 프라이버시 약속 {#the-privacy-promise}

> 모든 DNS 필터링은 기기에서 이루어져요. Lava는 여러분의 브라우징을 자사 서버로 라우팅하지 않으며, 여러분이 방문하는 도메인의 흐름을 절대 받지 않아요. 백엔드는 카탈로그 메타데이터, 사용자별 불투명 암호화 백업, 그리고 여러분이 보내기로 선택한 익명 진단 정보만 보관해요.

이 문장은 기준이 되는 약속이에요. 이 문서의 나머지 모든 내용은 이 문장과 일관되게 작성되어 있어요. 선택형 유료 등급을 결제하더라도 필터링이 서버로 옮겨지거나 Lava가 방문 도메인의 흐름을 받게 되지는 **않아요**. 어떤 기능이 서버와 관련될 때에는, 무엇이 **전송되지 않는지**를 문서에 명확히 적어 두었어요. 평소의 DNS 쿼리, 브라우징 기록, 그리고 모든 평문은 기기에 그대로 남아요. 전체 그림은 [백엔드 및 데이터 모델](../architecture/backend-and-data.md)을 참고하세요.

## 누구를 위한 앱인가요 {#who-it-is-for}

Lava는 직접 관리하는 수고 없이 더 안전한 브라우징을 원하는 누구나를 위해 만들어졌어요. 대상에는 비전문가 사용자도 일부러 포함했어요. 가족을 위해 보호 기능을 설정하는 부모님, 어르신, 그리고 DNS에 대해 전혀 신경 쓰고 싶지 않은 모든 분들이에요. 기본 경험은 그냥 작동해요. 보호 기능을 켜면 보수적인 차단 목록이 필터링을 시작하며, 계정도 필요 없어요. 동시에, 고급 사용자는 원할 때 더 깊은 설정(사용자 지정 차단 목록, 대체 리졸버)에 접근할 수 있어요.

전반적인 어조는 쉽고, 차분하며, 실용적이에요. 위험은 두려움이 아니라 비유로 표현해요.

## 핵심 원칙 {#core-principles}

- **프라이버시는 유료 기능이 아니라 우리의 기본 입장이에요.** 필터링은 로컬 판단이에요. Lava의 백엔드는 의도적으로 최소한으로 유지되며, 여러분의 평소 브라우징 도메인이나 DNS 이벤트 흐름을 절대 받지 않아요. 선택형 계정 백업은 [영지식](../architecture/accounts-and-backup.md) 방식이에요. 서버는 암호문과 비밀이 아닌 봉투 메타데이터만 저장해요.
- **핵심 보호 기능은 영원히 무료예요.** 보호 스위치, 기본 차단 목록 업데이트, 그리고 기본적인 로컬 카운트는 절대 제한되지 않으며 계정도 필요 없어요.
- **기기 내부에서 동작해요.** 보호 엔진은 전적으로 휴대폰 안에 있어요. DNS 분석, 도메인 평가, 상위 서버 전달이 모두 패킷 터널 익스텐션 내부에서 이루어지며, iOS의 익스텐션당 약 50 MiB 메모리 한도 안에서 동작해요. 차단 목록은 [소스 URL만 사용하는](../architecture/dns-filtering-and-blocklists.md) 방식을 따라요. 앱이 각 상위 목록을 직접 가져와 기기에서 분석하며, Lava는 서드파티 차단 목록 바이트를 호스팅하거나 제공하지 않아요.
- **결제는 맞춤 설정만 잠금 해제하며, 기본 안전은 절대 건드리지 않아요.** 위협 가드레일은 모든 차단 목록 위에 있는, 허용 불가능한 등급으로, 유료든 무료든 누구도 허용 목록에 넣을 수 없어요. 이는 판단 우선순위로 강제돼요. **위협 가드레일 > 로컬 허용 목록(허용된 예외) > 차단 목록 > 기본 허용.** (이 우선순위 슬롯은 연결되어 있으며 승인된 SHA-256 해시로 무결성이 검증돼요. 현재는 항목 없이 배포돼요.) 터널은 `isPaid`를 무시해요.
- **차분한 기본, 노력으로 얻는 깊이.** 기본 화면은 조용하고 안심을 주며, Soft Shield Guardian 마스코트와 두려움을 자극하지 않는 문구가 앞에 나서요. 더 풍부하고 기술적인 세부 정보는 찾아보면 볼 수 있지만, 절대 강요되지 않아요. 이 "차분한 기본, 노력으로 얻는 깊이" 철학은 **LavaTier** 깊이 모델(Floor / Window / Workshop)로 정형화되어 있어요. [디자인 시스템](../design-system/overview.md)을 참고하세요.

## 주요 기능 개요 {#high-level-capabilities}

- **로컬 DNS 필터링** — DNS를 분석하고, 컴파일된 스냅숏과 각 도메인을 대조해 평가하며, 허용된 쿼리를 상위 서버로 전달하는 패킷 터널 엔진이에요. 기기 DNS 폴백도 갖추고 있어요. [iOS 클라이언트](../architecture/ios-client.md)와 [DNS 필터링 및 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **엄선된 차단 목록, 소스 URL만 사용** — Lava는 상위 목록 URL과 승인된 해시만 게시해요. 기기가 목록 바이트를 직접 가져오고, 검증하고, 분석하며, Lava는 서드파티 차단 목록 바이트를 미러링하거나 제공하지 않아요. 배포되는 기본값은 **Block List Project Phishing + Scam**을 활성화해요(`AppConfiguration.lavaRecommendedDefaults`, `OnboardingDefaults.swift`에 정의됨). GPL 소스(HaGeZi, OISD)는 선택 사항이에요. [DNS 필터링 및 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **암호화된 DNS 전송** — DoH(관찰용 DoH3 주석 포함), DoT(연결 풀링, 재사용 및 갱신), DoQ(쿼리마다 새 연결)예요. 세 가지 모두 구현되어 있어요. Device DNS(네트워크 자체의 리졸버)가 배포되는 기본값이며, 암호화 프리셋은 선택 사항이에요(`AppConfiguration.lavaRecommendedDefaults`, `Sources/LavaSecCore/OnboardingDefaults.swift`에 정의됨). 내장 리졸버 프리셋(Google / Cloudflare / Quad9의 DoH 및 DoT 변형)은 무료예요. 완전한 사용자 지정 리졸버만 유료 잠금 해제 항목이에요. [DNS 필터링 및 차단 목록](../architecture/dns-filtering-and-blocklists.md)을 참고하세요.
- **허용된 예외(허용 목록)** — 차단 목록에 있더라도 허용하도록 도메인을 직접 추가할 수 있어요. 위협 가드레일은 여전히 우선해요. [제품 기능 개요](features.md)를 참고하세요.
- **Soft Shield Guardian** — Guard 탭, Live Activity, Dynamic Island에 등장하는 마스코트로, 7가지 표정 상태로 보호 상태를 표현해요. [디자인 시스템](../design-system/overview.md)을 참고하세요.
- **등급별 맞춤 설정(Lava Security Plus)** — 맞춤 설정을 잠금 해제하는 하나의 선택형 유료 등급이에요(더 큰 필터 규칙 예산 — 공유 기기 안전 가드레일 아래에서 Free 500K / Plus 2M 컴파일 규칙 —, 더 많은 허용/차단 도메인, 사용자 지정 차단 목록, 사용자 지정 DNS 리졸버). Plus는 항상 켜져 있는 가드레일을 절대 우회하지 않아요. 터널은 `isPaid`를 무시해요.
- **선택형 계정 및 백업** — Apple 또는 Google 로그인과, 종단 간 암호화된([영지식](../architecture/accounts-and-backup.md)) 설정 백업 및 복구 문구를 제공해요. 계정 삭제는 직접 할 수 있어요. 선택형 패스키 복구 슬롯도 **마찬가지로 영지식**이에요. 그 키는 인증기의 WebAuthn PRF로부터 기기에서 파생되며, 서버에 보관되는 비밀은 없어요. 기기에서의 프로덕션 준비는 Associated Domains / AASA 호스팅에 여전히 의존해요 **(예정)**. 계정은 선택 사항이며, 로그아웃 상태에서도 보호 기능은 완전히 작동해요.
- **로컬 전용 활동 및 보고서** — 기기 내 차단/허용 카운트, 터널 상태, 그리고 선택형 버그 보고서 번들이에요. 실행 중인 터널이 기기에 보관하는 데이터로 만들어지며, 유휴 상태일 때는 비어 있고 보호 중일 때는 실시간으로 채워져요. 평소의 도메인 기록은 기기를 떠나지 않아요. [제품 기능 개요](features.md)를 참고하세요.

## 플랫폼 {#platforms}

- **iOS — 출시됨.** Lava는 현재 iOS 앱이에요. 세 개의 번들이 하나의 App Group(`group.com.lavasec`)을 공유해요. 앱(`com.lavasec.app`), 패킷 터널 익스텐션(`.tunnel`), 위젯(`.widget`)이며, 여기에 공유 소스가 더해지고, 모두 공통 `LavaSecCore` 패키지 위에서 동작해요.
- **Android — 예정.** Android의 `VpnService` 위에서 네이티브 Kotlin / Jetpack Compose로 포팅하는 작업이 예정되어 있어요. 동일한 프라이버시 약속과, 동등성 테스트를 거친 핵심 필터링 동작을 그대로 가져갈 거예요. 아직 출시된 Android 앱 코드는 없어요.

안정적인 기능 id와 iOS/Android 계약에 대해서는 [플랫폼 동등성](platform-parity.md)을 참고하세요.
