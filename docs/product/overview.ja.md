# 製品概要 {#product-overview}

Lava Security へようこそ。このページはドキュメント全体の入り口です。Lava とは何か、何を約束するのか、そしてもっと詳しく知りたいときはどこを読めばいいのかを、短く・わかりやすくまとめています。

## Lava とは {#what-lava-is}

Lava Security は、プライバシーを最優先に考えた iOS アプリです。端末上で動く [NetworkExtension のパケットトンネル](../architecture/ios-client.md)を使って、DNS をこのスマートフォン上でローカルにフィルタリングします。危険だとわかっているドメインや、見たくないドメインをブロックしますが、その際にあなたのブラウジングを Lava のサーバー経由で流すようなことはしません。パケットトンネル（`LavaSecTunnel`、`NEPacketTunnelProvider` の一種）が、それぞれの DNS クエリを端末側で解析し、リクエストされたドメインを、コンパイル済みでメモリにマップされたフィルタースナップショットと照合し、許可されたクエリだけを上流へ転送します。あなたの通信が通過する Lava 運営のプロキシは存在しません。フィルタリングはあくまでローカルな判断で、あなたの端末上で行われます。

iOS がこれを「VPN」と呼ぶのは、アプリがシステム全体の DNS をフィルタリングできる唯一の手段がパケットトンネルだからです。ただし Lava がやっているのは **DNS／ブロックリストのフィルタリング**であって、通信の経路変更ではありません。できることの範囲については正直にお伝えします。Lava はローカルでの DNS ドメインフィルタリングであって、すべての悪意あるドメインや URL をブロックすると保証するものでは**ありません**。見えているのはドメインであってページのパスではないので、信頼できるホスト上にある 1 つの悪いページだけをブロックすることはできません。また、オンボーディングが終わった瞬間に保護が自動でオンになるわけでもありません。今まさに保護が有効かどうかは、アプリ内の **Guard** タブが正式な確認場所です。

## プライバシーに関する約束 {#the-privacy-promise}

> DNS のフィルタリングはすべて端末上で行われます。Lava があなたのブラウジングを自社サーバー経由で流すことは一切なく、あなたが訪れたドメインの流れを受け取ることもありません。バックエンドが持っているのは、カタログのメタデータ、ユーザーごとに中身の見えない暗号化されたバックアップ、そしてあなたが送ることを選んだ匿名化された診断情報だけです。

この一文が基準（カノニカル）です。このドキュメントの他のすべての内容は、これと矛盾しないように書かれています。任意の有料プランにお金を払っても、フィルタリングがサーバー側に移ったり、あなたが訪れたドメインの流れが Lava に渡ったりすることは**ありません**。ある機能がサーバーに触れる場合は、何が**送られない**のかをドキュメントではっきり書いています。日々の DNS クエリ、ブラウジング履歴、平文のデータはすべて端末上にとどまります。全体像については[バックエンドとデータモデル](../architecture/backend-and-data.md)をご覧ください。

## 誰のためのものか {#who-it-is-for}

Lava は、面倒な管理なしに、より安全なブラウジングを求めるすべての人のために作られています。対象には、技術にあまり詳しくない人もあえて含めています。家族のために保護を設定する保護者、年配の方、そして DNS のことなんてまったく考えたくない人。デフォルトの体験は、ただ動くだけです。保護をオンにすれば、控えめなブロックリストがフィルタリングを始めます。アカウントは不要です。同時に、もっと深い操作（カスタムブロックリストや別のリゾルバー）を使いたいパワーユーザーは、必要なときにそこへ手を伸ばせます。

全体を通しての語り口は、わかりやすく、落ち着いていて、実用的です。危険は、恐怖ではなくたとえ話として描いています。

## 基本理念 {#core-principles}

- **プライバシーは立ち位置であって、有料機能ではありません。** フィルタリングはローカルな判断です。Lava のバックエンドは意図的に最小限で、あなたの日々のブラウジングドメインや DNS イベントの流れを受け取ることは決してありません。任意のアカウントバックアップは[ゼロ知識](../architecture/accounts-and-backup.md)です。サーバーが保存するのは暗号文と、秘密ではない封筒（エンベロープ）メタデータだけです。
- **基本の保護はずっと無料。** 保護スイッチ、デフォルトブロックリストの更新、基本的なローカルカウントは、制限されることもアカウントを求められることも決してありません。
- **端末上で完結。** 保護エンジンはすべてスマートフォンの中にあります。DNS の解析、ドメインの評価、上流への転送はすべてパケットトンネル拡張機能の中で行われ、iOS の拡張機能あたり約 50 MiB というメモリ上限の範囲で動きます。ブロックリストは[ソース URL のみ](../architecture/dns-filtering-and-blocklists.md)というモデルに従います。アプリが各上流リストを直接取得してローカルで解析するので、Lava がサードパーティのブロックリストのバイト列をホストしたり配信したりすることはありません。
- **支払いで開放されるのはカスタマイズだけ。基本の安全性は決して対象外です。** 脅威ガードレール — どのブロックリストよりも上位にある、許可（allowlist）できない層で、有料か無料かに関わらず誰も許可リストに入れられない — は、判断の優先順位によって守られています。**脅威ガードレール ＞ ローカルの許可リスト（許可する例外）＞ ブロックリスト ＞ デフォルト許可。**（この優先スロットは配線済みで、受理済みの SHA-256 ハッシュで整合性チェックされていますが、現時点では中身（エントリ）なしで出荷されています。）トンネルは `isPaid` を無視します。
- **核は落ち着いて、深さは求めた人に。** デフォルトの画面は静かで安心感があり、Soft Shield Guardian マスコットと、恐怖をあおらないコピーが前面に立ちます。もっと豊かで技術的な詳細は、探しに行けば手に入りますが、決して押し付けられることはありません。この「落ち着いた核、求めた人への深さ」という考え方は、**LavaTier** の深度モデル（Floor／Window／Workshop）として形になっています。[デザインシステム](../design-system/overview.md)をご覧ください。

## 主な機能（概観） {#high-level-capabilities}

- **ローカル DNS フィルタリング** — DNS を解析し、各ドメインをコンパイル済みスナップショットと照合し、許可されたクエリを上流へ転送する（デバイス DNS フォールバックつき）パケットトンネルエンジンです。[iOS クライアント](../architecture/ios-client.md)と [DNS フィルタリングとブロックリスト](../architecture/dns-filtering-and-blocklists.md)をご覧ください。
- **用意されたブロックリスト、ソース URL のみ** — Lava が公開するのは上流リストの URL（に加えて、キャッシュ識別と監査のための参考ハッシュ）だけです。端末が各リストを TLS 経由で取得し、サイズ／ルールの上限のもとでローカルに解析します。Lava がサードパーティのブロックリストのバイト列をミラーしたり配信したりすることは決してありません。コミュニティのリストはハッシュで固定されていません — TLS と用意された URL が整合性の境界です — 一方で Lava の脅威ガードレール層はハッシュで強制され続けます。出荷時のデフォルトでは **Block List Basic** が有効になります（`AppConfiguration.lavaRecommendedDefaults`、`OnboardingDefaults.swift` で定義）。HaGeZi、OISD、AdGuard、1Hosts といったコピーレフトのソースはオプトインです。[DNS フィルタリングとブロックリスト](../architecture/dns-filtering-and-blocklists.md)をご覧ください。
- **暗号化された DNS トランスポート** — DoH（観測用の DoH3 アノテーションつき）、DoT（コネクションをプールし、再利用・更新）、DoQ（クエリごとに新しいコネクション）。3 つすべて実装済みです。Device DNS（ネットワーク自身のリゾルバー）が出荷時のデフォルトで、暗号化プリセットはオプトインです（`AppConfiguration.lavaRecommendedDefaults`、`Sources/LavaSecCore/OnboardingDefaults.swift` で定義）。内蔵のリゾルバープリセット（Google／Cloudflare／Quad9 の DoH と DoT バリアント）は無料です。完全にカスタムなリゾルバーだけが有料で開放されます。[DNS フィルタリングとブロックリスト](../architecture/dns-filtering-and-blocklists.md)をご覧ください。
- **許可する例外（許可リスト）** — ブロックリストに載っていても許可したいドメインを手動で追加できます。それでも脅威ガードレールが優先されます。[製品機能の概要](features.md)をご覧ください。
- **Soft Shield Guardian** — Guard タブ、Live Activity、Dynamic Island に登場するマスコットで、7 つの表情の状態で保護状態を表します。[デザインシステム](../design-system/overview.md)をご覧ください。
- **段階的なカスタマイズ（Lava Security Plus）** — カスタマイズを開放する 1 つの任意の有料プランです（より大きなフィルタールールの上限 — 無料は 50 万、Plus は 200 万のコンパイル済みルール、共通の端末安全ガードレールのもとで — そして、より多くの許可／ブロックドメイン、カスタムブロックリスト、カスタム DNS リゾルバー）。Plus でも、常時オンのガードレールを回避することは決してありません。トンネルは `isPaid` を無視します。
- **任意のアカウントとバックアップ** — Apple または Google でのサインインと、エンドツーエンドで暗号化された（[ゼロ知識](../architecture/accounts-and-backup.md)）設定バックアップ、そしてリカバリーフレーズ。アカウント削除はセルフサービスです。任意のパスキーによるリカバリースロットも**同じくゼロ知識**です。その鍵は認証器の WebAuthn PRF から端末上で導出され、サーバー側に秘密を持ちません。端末上での本番運用への対応は、依然として Associated Domains／AASA のホスティング次第です **(予定)**。アカウントは任意で、サインアウトしたままでも保護は完全に機能します。
- **ローカルのみのアクティビティとレポート** — 端末上のブロック／許可カウント、トンネルの健全性、そしてオプトインの不具合レポートのまとまり。これらは、動作中のトンネルが端末上に保持しているデータから作られます。アイドル時は空っぽで、保護中はリアルタイムで更新されます。日々のドメイン履歴が端末から出ていくことはありません。[製品機能の概要](features.md)をご覧ください。

## プラットフォーム {#platforms}

- **iOS — 出荷済み。** Lava は今のところ iOS アプリです。3 つのバンドルが 1 つの App Group（`group.com.lavasec`）を共有します。アプリ本体（`com.lavasec.app`）、パケットトンネル拡張機能（`.tunnel`）、ウィジェット（`.widget`）に加えて、共有ソース、そして共通の `LavaSecCore` パッケージです。
- **Android — 予定。** Android の `VpnService` 上に、ネイティブな Kotlin／Jetpack Compose で移植する計画があります。同じプライバシーの約束を引き継ぎ、コアのフィルタリング挙動はパリティ（同等性）をテストします。Android のアプリコードはまだ出荷していません。

安定した機能 id と iOS／Android の取り決めについては、[プラットフォーム・パリティ](platform-parity.md)をご覧ください。
